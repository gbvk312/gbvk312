# Profile Maintenance Rules: The Automation Architect

These rules MUST be followed when modifying the GitHub profile README or related workflows to maintain the **Automation Architect** persona.

## Profile README Components

- **GitHub Stats, Top Languages & Trophies**:
    - Always verify that the Vercel URLs are functional.
    - If `github-readme-stats.vercel.app` is down (503/504), prefer the `anuraghazra1` mirror: `github-readme-stats-anuraghazra1.vercel.app`.
    - If images appear broken, use `github-profile-trophy.vercel.app` as the stable fallback.
    - Never commit changes to these URLs without using the `browser_subagent` or `read_url_content` to verify they render correctly.

- **Snake Animation & Interactive Elements**:
    - The animated snake is generated by the `snake.yml` workflow and stored in the `output` branch.
    - To verify: Check if `github-contribution-grid-snake.svg` exists on the `output` branch.
    - Maintain the "Classic" theme for the snake animation unless otherwise requested.

- **Achievements & Metrics (Legacy/Secondary)**:
    - These are generated by the `metrics.yml` workflow and stored in the `metrics` branch.
    - If images appear broken, fetch the `metrics` branch to ensure the SVG files exist and are up-to-date.
    - Standardize on `raw.githubusercontent.com` URLs for all static SVGs.

## Verification Requirements

1. **Visual Validation**: After any edit to `README.md`, run a `browser_subagent` task to capture a screenshot of the profile to ensure everything aligns and renders.
2. **Link Health**: If you add new links to projects or social media, verify them with `read_url_content`.

## Workflow Integrity

- Do not modify `.github/workflows/metrics.yml` or `.github/workflows/profile-validator.yml` without ensuring the `METRICS_TOKEN` secret is correctly referenced.
- If the `profile-validator.yml` fails, it is a priority to fix the broken component immediately.
